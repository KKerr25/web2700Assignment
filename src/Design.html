Introduction

Good visual design and structure are critical in web development because they create a user-friendly experience that is accessible to all. Proper design ensures that content is readable, visually appealing, and navigable for users with varying needs, including those relying on assistive technologies. Strong semantic structure improves usability, search engine optimization (SEO), and helps developers maintain and extend code easily.

Visual Design

Color

  • Primary Color: The primary color I chose is blue because it conveys trust, professionalism, and calmness.

  • Secondary Color: The secondary color is white, chosen for its clean and neutral qualities.

  • Tertiary Color: The tertiary color is a darker blue, selected for its complementary qualities with the primary blue while adding depth and contrast.

  • Selection Process: To choose these colors, I considered color psychology and tested combinations for aesthetic appeal and accessibility using tools like Adobe Color and contrast checkers.

  • Application of Colors: Blue is used for buttons and headings, providing a strong and professional emphasis. White is applied as the background to ensure a clean, uncluttered interface that makes other elements stand out. The darker blue is used for secondary accents, such as borders, hover states, or text, to emphasize important content and maintain a cohesive design.

  • Contrast Ratios: The contrast between the darker blue elements and the white background exceeds the WCAG 2.1 AA contrast ratio requirement of 4.5:1 for text and 3:1 for larger elements, ensuring accessibility for users with visual impairments.

Typography

  • Body Font: The body font chosen is "Open Sans," a sans-serif font known for its legibility across devices and sizes.

  • Header Font: For headers, I opted for "Roboto Bold."

  • Sizing, Weight, and Colors: Font sizes are 16px for body text and larger sizes for headings to establish a clear hierarchy. Weights range from regular for paragraphs to bold for emphasis, ensuring readability. Fonts are dark grey against a white background, ensuring high contrast.

Layout/Navigation

Visual Layout

The layout follows a traditional hierarchy, with navigation at the top, followed by a clear header, main content, and footer. Elements are arranged logically to match reading order, starting from left to right and top to bottom. This structure ensures that users can quickly locate what they're looking for without confusion.

Navigation

Navigation includes clear hover states, focus states, and active states to ensure users—especially those relying on keyboard navigation—can easily track where they are. Visual indicators such as underlining or color changes are used to provide accessibility and intuitive interaction.

Content

Content Structuring

The content is structured with semantic HTML elements like header, main, section, aside, and footer to ensure clarity and alignment with accessibility standards.

Responsiveness

The site is fully responsive and supports resizing without compromising readability. Text zoom up to 200% is supported without breaking the layout or hiding important information.

Bootstrap

Bootstrap was used to quickly prototype and implement responsive grid systems, ensuring consistent spacing and alignment across devices.

Structural Design

Accessibility Features

  • ARIA: ARIA roles and labels are used to provide context for screen readers, such as aria-labelledby on sections within main.
  • Semantic Elements: Proper semantic tags are used for navigation (nav), headers (header), and content (main), ensuring logical structure.
  • Descriptive Link Text: All links are descriptive, avoiding generic text like "Click here."
  • Focus Styles: Clearly visible focus styles are provided for keyboard users, such as outlines or color changes.
  • Skip Navigation Link: A skip link is included to let users jump directly to the main content.
  • Responsive Design: Media queries and flexible layouts ensure accessibility across various screen sizes.
  • Alt-Text: Alt-text descriptions are used for all images to convey their content or purpose.
  • Testing: The site was tested with accessibility tools like Lighthouse, Axe, and screen readers to ensure compliance and a good user experience.
